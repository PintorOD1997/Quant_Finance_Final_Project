Abstract

Este estudio establece un enfoque de opciones compuestas para evaluar los proyectos de inversin en I+D farmacutica en presencia de incertidumbres tcnicas y econmicas:

  • La incertidumbre tcnica se modela como un salto de Poisson que permite el fracaso y, por tanto, el abandono del desarrollo del frmaco.
  • La incertidumbre econmica se modela como un proceso de difusin estndar que incorpora choques al alza y a la baja.

La aplicacin prctica de este mtodo se enfatiza a travs de un anlisis de casos. Demostramos que ambas incertidumbres tienen un impacto positivo en el valor de la opcin de I+D. Adems, a partir del anlisis de sensibilidad, descubrimos que la sensibilidad de la opcin con respecto a la incertidumbre econmica y al coste de introduccin en el mercado disminuye cuando la incertidumbre tcnica aumenta. </font>

1. Introduction

La valoracin de opciones reales (ROV) es una buena alternativa para el campo farmacutico, ya que este enfoque puede aportar una mayor evaluacin flexible de las oportunidades futuras inherentes al proceso de innovacin.

La farmacutica interpreta la inversin en el proyecto como una opcin para expandir ms proyectos. En este caso particular, el xito en el primer proyecto, dara pie a un segundo proyecto; consecuentemente de la misma manera, el xito en el segundo proyecto volvera conveniente el entrar a un tercer proyecto, y as sucesivamente hasta plantearse la oportunidad de entrar a un cuarto proyecto. Todo esto le da al proyecto la forma de una call option compuesta.

Las opciones compuestas son combinaciones de opciones, donde el ejercicio de una opcin abre otra opcin, en la teora de las opciones reales, las aplicaciones de las opciones compuestas se encuentran en varios proyectos industriales, pero son ms relevantes para los productos farmacuticos donde el proyecto ofrece la opcin real de realizar ms investigaciones o de iniciar la implementacin de los resultados.

Para este proyecto partimos de un modelo bastante simple, el call de una call option con dos strike prices y dos fechas de maduracin.

Caso de estudio 1¶

Se utiliza la teora de opciones reales para evaluar cada una de las fases (3) por la que pasar un nuevo frmaco.

Se utilizarn las opciones reales al tratar cada fase como un nuevo proyecto en el que se evala si vale la pena invertir dependiendo de los resultados del proyecto (o etapa) anterior, y en donde se toma en cuenta el riesgo de fracaso del frmaco en cualquiera de las etapas al combinar una funcin Poisson (incertidumbre tcnica) y un proceso de difusin (incertidumbre econmica).

El valor del proyecto es incierto durante las diferentes etapas, pero asumimos que el valor sigue un proceso de salto normal de difusin, teniendo dos caminos de incertidumbre:

  • el riesgo de difusin, que incluye las fluctuaciones aleatorias tanto positivas como negativas
  • el arribo de sucesos "mayores" que abruptamente pueden afectar aumentando o disminuyendo el valor del proyecto

De acuerdo a la frmula del precio, es la misma que la frmula de opcin compuesta de Geske, dando que el fallo tcnico no ocurre durante los intrvalos.

Para probar nuestro modelo en un ajuste de negocio prctico, lo aplicaremos en una compaa de Europa enfocada al rea de oncologa, la cual quiere sacar una nueva lnea de tratamientos contra el cncer y donde tiene dos decisiones de inversin:

  • Entrar a la fase 3
  • Lanzar el medicamento

Para esto ambos casos se analizarn con la frmula de opciones compuestas.

def Call(S,K,T,r,l, sigma):
    d1 = (np.log(S/K) + (r+l + 0.5 * sigma ** 2) * T)/ (sigma * np.sqrt(T))
    #d2 = (np.log(S/K) + (r+l - 0.5 * sigma ** 2) * T)/ (sigma * np.sqrt(T))
    d2 = d1-sigma*np.sqrt(T)
    Call = np.exp(-l*T)*(S*np.exp(l*T) * si.norm.cdf(d1,0.0,1.0) - K * np.exp(-r * T) * 
                         si.norm.cdf(d2,0.0,1.0))
    return Call


def Vopt(S,K,t,r,l,sigma):
    """
    Funcin Vopt
    Clculo del Valor a partir del cul se vuelve rentable trabajar en la siguiente etapa
    del proyecto. (Seccin en construccin)
    Minimiza un Call cuyo valor de ejercicio es el valor definido para la siguiente etapa.
    """
    tau = t[1]-t[0]
    bounds = [[0, None]]
    apply_constraint1 = lambda S: Call(S, K[0], tau, r,l, sigma) - K[1] # Tiene que ser igual a la X siguiente
    my_constraints = ({'type': 'eq','fun': apply_constraint1})
    a = minimize(Call, S, bounds=bounds, constraints=my_constraints, method='SLSQP', args=(K[0], tau,
                                                                                           r,l, sigma))
    return a.x[0] #precio justo de la opcion

def hk(V,Vopt,r,l,sigma,T):
    """
    Funcin hk
    Esta funcin obtiene el valor de h para opciones compuestas. Equivalente al clculo de
    D1 en Black-Scholes.
    """
    return (np.log(V/Vopt) + (r+l + 0.5 * sigma ** 2) * T)/ (sigma * np.sqrt(T))

def lk(h,sigma,T):
    """
    Funcin lk
    Obtiene el valor de l a partir del valor calculado previamente de h. Equivalente al clculo
    de D2 en Black-Scholes.
    """
    return h-sigma*np.sqrt(T)
def corrMatrix(T):
    """
    Funcin corrMatrix
    Esta funcin obtiene la matriz de correlacin simtrica para N tiempos 
    Los valores de correlacin son la raz cuadrada del cociente de los tiempos.
    """
    cov = np.ones((len(T),len(T)))
    for i in range(len(T)):
        for j in range(len(T)):
            if i<=j:
                cov[j,i] = np.sqrt(T[i]/T[j])
            else:
                cov[j,i] = np.sqrt(T[j]/T[i])
    return cov

def NcompoundOption(S,K,tiempo,r,l,sigma):
    """
    Funcin NcompoundOption
    Generalizacin del modelo de N-anidadas opciones compuestas.
    """
    N = len(tiempo) 
    suma = np.zeros(N) 
    hs = np.zeros(N)
    ls = np.zeros(N)
    Vo = np.zeros(N)
    ehepaar = np.array(K[:2]) # TIene que agarrar slice de los K's recibidos
    Vo[0] = Vopt(S,ehepaar,tiempo,r,l,sigma)
    # Primer ciclo, clculo de ls y hs
    for j in range(len(tiempo)):
        if j == len(tiempo)-1: #En el tiempo final, el valor ptimo de ejercicio es igual a I2
            Vo[j] = K[j]
            hs[j] = hk(S,Vo[j],r,l,sigma,tiempo[j])
            ls[j] = lk(hs[j],sigma,tiempo[j])
        else:
            Vo[j] = Vopt(S,ehepaar,tiempo,r,l,sigma)
             #Tiene que agarrar Slice de Ehepaar
            hs[j] = hk(S,Vo[j],r,l,sigma,tiempo[j])
            ls[j] = lk(hs[j],sigma,tiempo[j])
            ehepaar = np.array([Vo[j],K[j]])
    # Segundo ciclo, distribuciones y suma
    for j in range(len(tiempo)):
        dist = mvn(mean=np.zeros(N+1-(j+1)),cov=corrMatrix(tiempo[0:N+1-(j+1)]))

        if N+1-(j+1) == 1:
            suma[j] = K[j]*np.exp(-(r+l)*tiempo[j])*dist.cdf(ls[j-1])
        else:
            #print(ls[j:N+1-(j)])
            suma[j] = K[j]*np.exp(-(r+l)*tiempo[j])*dist.cdf(ls[j:])
    dist = mvn(mean=np.zeros(N),cov=corrMatrix(tiempo))
    #print(S*dist.cdf(hs),-suma)
    return S*dist.cdf(hs)-suma.sum()
